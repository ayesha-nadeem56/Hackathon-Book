## Follow-ups and Risks

- **Risk 1**: Mermaid diagram complexity — if diagrams become too large, Docusaurus may render them slowly or clip them. Mitigation: keep each diagram to ≤8 nodes.
- **Risk 2**: SDF/XML examples in prose — angle brackets must be backtick-wrapped; a missed bracket will break the MDX build. Mitigation: encode this as explicit acceptance check; validate with build.
- **Risk 3**: Unity ROS-TCP-Connector version drift — the integration details change between ROS 2 versions. Mitigation: describe the concept and data flow rather than exact API calls; note Humble compatibility.
